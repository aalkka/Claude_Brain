# 골든셋 측정 절차 — hit@8

검색 성능을 골든셋으로 측정·회귀한다. 지표 = **hit@8** = (정답 노트가 top-8 후보에 포함된 질의 수) / 전체. 합격 = **≥ 80%**.

## 1. 골든셋 작성
`_eval/*.jsonl`, 각 줄 = `{"q":"한국어 질의","a":"정답 노트 상대경로"}`. **골든은 배포에 없음 — 각 사용자가 자기 볼트로 작성**한다.
- **literal형**: 노트의 고유명사·헤딩어를 그대로 쓴 질의(grep/lexical 강점 구간).
- **패러프레이즈형**: 고유명사·헤딩어를 뺀 의미 질의(벡터/하이브리드 검증, **실사용 근사**).
- 배분 권장: 지식질의 · 세션회상형 · _ref형("전에 이 버그 어떻게 풀었지") · PDF내용형.

> ⚠ **함정: literal만 재면 순환측정.** literal 질의는 grep 홈그라운드(고유명사 공유)라 grep이 거의 100% — "검색이 좋다"가 아니라 "질의가 grep에 유리"였을 뿐. **패러프레이즈 골든을 반드시 병행**해야 각 arm의 강점과 하이브리드의 상보성이 드러난다. (이 함정을 놓쳐 벡터 불요로 오판할 뻔한 게 빌드 중 실측됨 — `results-hybrid.md`.)

## 2. 측정
```
py -3 3_시스템/search.py --eval <골든파일> --k 8
```
→ **lexical / vector / hybrid 3자 hit@8** 자동 출력 + miss 목록(정답의 실제 순위).

## 3. 판정
- 패러프레이즈 골든에서 **하이브리드가 각 arm 단독을 능가**하면 하이브리드 유지(현 채택 구성).
- hit@8 < 80%면 임베딩 티어 상향(`config.json` `embed_model` 교체 + `search.py --rebuild`) 후 재측정.

## 4. 회귀 (weekly)
`weekly-review`가 골든 샘플을 재측정(3자) → 하이브리드 낙폭 시 보고. 재측정 트리거: 코퍼스 급증(수백 노트↑) · 이전 대비 하락 · 임베딩 모델 교체.
