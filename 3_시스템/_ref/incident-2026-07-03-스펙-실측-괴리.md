# 인시던트: 설계 스펙 vs 실측 괴리 (Phase 0 빌드)

## 증상
Phase 0 빌드 중 설계 스펙대로 만들었으나 5건이 실제 환경서 작동 안 하거나 오작동.

## 근본원인
설계가 Claude Code 네이티브 동작·Windows/PowerShell 특성을 실측 없이 가정.

1. **스킬 미로드** — 설계 `3_시스템/skills/`. Claude Code는 `.claude/skills/`(또는 `~/.claude/skills/`)만 탐색.
2. **secret 오탐** — pre-commit `sk-` 순수 부분문자열 → Dataview CSS `task-`의 `sk-` 매칭 → 정상 커밋 차단.
3. **sensitive 오탐** — `^sensitive:...true` 앵커 없어 문서 산문("sensitive: true 시…"=conventions.md)을 frontmatter로 오인.
4. **PowerShell 변수명 흡수** — `"$noteCount개"`서 한글 `개`가 변수명 일부로 흡수(유니코드 letter) → 빈 출력.
5. **옵션R 메모리 고아화** — autoMemoryDirectory를 빈 새 dir로 리다이렉트 시 기존 메모리(옛 경로) 안 읽힘.
6. **★배포 치명: PS1 인코딩** — Windows PowerShell **5.1은 .ps1을 시스템 ANSI(cp949)로 읽음**(UTF-8 아님). 훅 파일이 UTF-8-no-BOM + 한글 경로(`3_시스템`)·`{{` 포함 → 5.1서 파서 에러(session-start 아예 미실행) 또는 런타임 문자열 리터럴 깨짐(session-end는 커밋되나 로그경로·`1_수집` 제외 오염). pwsh 7은 UTF-8 기본이라 개발 중엔 안 잡힘 → **훅은 `powershell`(5.1)로 도는데 검증을 pwsh로 해서 놓칠 뻔.** 모든 Windows 사용자 히트.
7. **★배포 치명: pre-commit 미배포** — `.git/hooks/`는 git 추적 안 됨 → clone·재clone 시 secret 게이트 소멸. 설계가 위치(.git/hooks)만 적고 배포 경로 누락.
8. **secret 정규식 하이픈 갭** — `sk-[A-Za-z0-9]{6,}`는 하이픈 못 넘어 실제 최신 키(`sk-ant-`·`sk-proj-`·`sk-live-`) 놓침(false negative=위험). Phase 0 감사서 실키형 테스트로 적발.
9. **데이터 파일 read 인코딩(#6과 별건)** — session-start.ps1이 `Get-Content recent.md/open-loops.md -Raw` 인코딩 미지정. PS5.1 기본 read=cp949 → BOM없는 UTF-8 데이터를 cp949로 오독 → **주입 컨텍스트 모지바케.** #6은 *스크립트 파일*(파서), 이건 *데이터 파일*(read) — 층이 다름. line4 `OutputEncoding=UTF8`은 출력만 고쳐 read 단계 이미 깨진 뒤라 무효. SessionStart:resume 실발화 로그서 건희님이 적발.

## 해결
1. `git mv 3_시스템/skills .claude/skills` — 이관 즉시 라이브 로드 확인.
2. secret 정규식 → `\b(sk-|ghp_)[A-Za-z0-9]{6,}|PRIVATE KEY`(단어경계+길이).
3. sensitive 정규식 → `^sensitive:[[:space:]]*true[[:space:]]*$`(줄끝 앵커) + HEAD 가드(최초커밋).
4. `"${noteCount}개"`(중괄호 경계).
5. 리다이렉트 전 기존 메모리 2파일을 새 dir로 복사(연속성).
6. **한글 포함 .ps1은 UTF-8 BOM으로 저장**(`Set-Content -Encoding utf8BOM`). 5.1이 BOM 보면 UTF-8로 읽음. 검증: `powershell -File`(5.1)로 실행해 파싱+런타임 확인(pwsh 아님). BOM 후 5.1서 헬스라인 출력·hooks.log 정상 실증.
7. **pre-commit → `3_시스템/hooks/`(추적) + `git config core.hooksPath 3_시스템/hooks`**(설치 스텝서 실행, core.hooksPath는 미배포 로컬config). 검증: 실키형 커밋 차단.
8. **정규식 `(\bsk-[A-Za-z0-9_-]{16,}|\bghp_[A-Za-z0-9]{16,}|\bAKIA[0-9A-Z]{16}|PRIVATE KEY)`** — 하이픈 허용+16자+단어경계. 실키 잡고 `task-`·짧은CSS 제외.
9. **`Get-Content` 데이터 read에 `-Encoding UTF8` 명시**(session-start.ps1 line20-21). PS5.1은 read 기본이 cp949라 BOM없는 UTF-8 데이터엔 필수. (line13 CLAUDE.md read는 `\{\{` ASCII 매칭이라 오독돼도 무해 — 미수정.) 검증: 5.1서 recent/open-loops 한글 정상 주입.

## 재발방지
- **Claude Code 네이티브 경로·동작은 스펙 신뢰 말고 실측**(스킬/훅/메모리 위치).
- 훅 정규식은 오탐/오누락 양방향 테스트(예: `task-` vs `sk-key`).
- PowerShell 문자열 보간서 한글 인접 변수는 `${}` 경계 필수.
- 리다이렉트·이관은 기존 데이터 연속성 먼저 확보.
- **한글/비ASCII .ps1 = UTF-8 BOM 필수. 검증은 실제 런타임(5.1)로, pwsh 7로 하면 인코딩 버그 놓침.** (교훈: 개발툴≠런타임툴이면 런타임툴로 검증.)
- **PS5.1 인코딩은 2층: ①스크립트 파일=BOM(#6) ②데이터 read=`-Encoding UTF8`(#9).** 둘 다 필요, 한쪽만 고치면 나머지서 깨짐. 훅이 UTF-8 파일 읽으면 항상 `-Encoding UTF8`.
