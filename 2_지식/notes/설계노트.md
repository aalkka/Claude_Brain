# 외부뇌 설계노트 (통합)

> **이 문서는 무엇인가.** 외부뇌(External Brain)가 **왜 지금의 모양인지**를 설명하는 단일 설계 문서다. [[사용자설명서]]가 "어떻게 쓰나"라면, 이 문서는 "왜 이렇게 지어졌나"다. 사용자설명서를 읽다 더 깊이 알고 싶은 부분에서 이 문서의 해당 절로 넘어오면 된다.
>
> **as-built 기준.** 아래는 실제로 제작된 시스템을 서술한다. 설계 과정에서 구상됐다가 잘려나간 정교한 장치들은 [[#10. DEFERRED 창고|§10 DEFERRED 창고]]에 복원 조건과 함께 보존한다. "지금 있는 것"과 "잘라둔 것"을 섞지 않는다.

---

## 1. 목적 — 세 가지

외부뇌는 개인 1인 지식관리 시스템이다. Obsidian 볼트(마크다운) 위에서 Claude Code가 지식을 쌓고 검색하고 되살린다. 세 목적을 위해 존재한다.

1. **토큰 최적화** — 불필요한 토큰을 없애고, 필요한 작업을 최소 토큰으로 완수한다. (입력토큰×입력단가 + 출력토큰×출력단가를 낮춘다.)
2. **맥락 복원** — 매번 긴 프롬프트로 설명하지 않아도, 이전 흐름과 사용자 정보를 자동으로 되살려 대화한다.
3. **답변 질 향상** — 답변에서 쓸모없는 부분을 제거하고, 근거 있는 구조화된 답을 만든다.

이 세 목적이 모든 설계 결정의 저울이다. 어떤 장치가 셋 중 하나에 직접 지렛대가 되지 못하면, 그것은 기둥이 아니라 가지다([[#9. 설계 원칙|§9 설계 원칙]]).

## 2. 왜 다시 지었나 — 구 시스템의 두 뿌리 문제

외부뇌는 처음부터 이 모양이 아니었다. 전신은 **Claude_Obsidian**(Cowork 기반, 볼트 `C:\Claude_Obsidian`)이었다. 사상과 골격(원자 노트·프론트매터·MOC·링크=메모리·3목적)은 견고했고 실제로 잘 작동했다. 문제는 골격 위에 얹힌 방어·최적화 장치들이 서로 얽히며 비대·불안정해진 것이었다. 사용자의 재설계 동기: *"기본 뼈대에 추가사항이 더해지며 계획안보다 비대해지고 그로 인해 불안정해짐. 해결 불가한 문제를 여럿 안고 있음."*

전수 진단 결과, 11종의 반복·치명 문제가 **두 개의 뿌리**로 수렴했다.

### 뿌리 ① — 샌드박스 ↔ PC 이중 실행 환경
Cowork(샌드박스)와 PC(Obsidian·git·스케줄러)가 같은 파일·같은 `.git`을 각자 만졌다. 여기서 치명 문제들이 파생됐다.
- **파일 꼬리 절단**: 쓰기 완료 전 외부 프로세스 경합으로 .md 뒷부분이 잘림. "모든 .md는 bash로만 쓴다"로 회피했으나 근본 해결은 아니었다.
- **git 인덱스 경합**: 두 주체가 같은 `.git/index`를 조작 → lock 충돌. "Claude git 쓰기 전면 금지"로 봉합했으나, 대가로 **매 세션 사용자가 수동 커밋**해야 루프가 닫혔다. 자동화 시스템이 마지막 1미터를 사람에게 떠넘긴 셈.
- **샌드박스↔PC stale**: PC에서 git 복구해도 샌드박스는 구본/손상본을 계속 봄.

### 뿌리 ② — 방어 규약의 누적
문제가 터질 때마다 CLAUDE.md·conventions에 규약을 얹었다. 그 결과:
- **기본 로드 토큰 단조 증가**: 규약 누적으로 455 → 1,575토큰. 상한 관리에 매 세션 주의를 소모.
- **설계 ≠ 실작동**: 기능을 "노트로 존재"시키는 것과 "매 세션 발동 경로에 연결"하는 것이 분리 → 죽은 규약 발생. 발동을 보장하려 또 다른 장치(트리거 로그)를 얹는 악순환.
- **방어 규약 비대·복잡**: 한 방어가 새 취약성을 낳는(bash 강제 → sed 편집 불안정) 얽힘.

### 재설계의 방향
두 뿌리를 정면으로 겨냥한다.
- **뿌리 ① → 실행 환경 단일화.** Cowork를 버리고 **Claude Code**로 이전. Claude가 볼트에 직접 읽고 쓰는 단일 주체 모델 → 샌드박스↔PC 분리가 사라지고 꼬리 절단·git 경합·stale·수동 개입이 대부분 소멸.
- **뿌리 ② → 규약을 설계 시점에 최소 코어로 확정.** 사후 누적이 아니라, 기본 로드를 구조적으로 작게 고정하고([[#3.3 계층 로드|§3 계층 로드]]) 강제는 규약이 아니라 **결정론적 훅**으로([[#7. 훅 3 — 결정론적 보증층|§7 훅]]) 옮긴다.

구 시스템의 문제 12건(11종 + inbox 손실 버그)은 신 설계에서 전수 대응됐고, 그중 대부분이 봉합이 아니라 뿌리 제거(ROOT)다. 봉합책(bash 강제·git 금지 A규칙)이 낳던 부작용(수동개입·sed 불안정)은 뿌리를 없애자 **봉합 자체가 불필요**해졌다.

## 3. 핵심 아키텍처

### 3.1 Obsidian + Claude Code, 디커플
- **마크다운 = 진실 원본.** 모든 지식은 사람이 읽을 수 있는 .md. 인덱스·임베딩은 언제든 재구축 가능한 파생물(손실돼도 무해).
- **Claude Code = 단일 쓰기 주체.** 볼트 폴더를 직접 읽고 쓴다. 이중 환경 경합 소멸(뿌리 ①).
- **Obsidian = 뷰어.** 그래프·Dataview 대시보드로 사람이 같은 지식을 본다. 시스템은 Obsidian 없이도 동작한다(Obsidian에 임베드하지 않음). 3rd-party 플러그인 의존 = **Dataview 하나뿐**(라이브 대시보드, Claude 토큰 0, 없어도 시스템 작동).

### 3.2 폴더 구조 — 시인성 3분류
누가 읽고 쓰는가로 폴더를 나눈다.
```
Claude_Brain/
├── CLAUDE.md              # 루트엔 이것만(+배포 문서). Claude Code가 루트서 로드
├── 1_수집/_inbox/          # ① 사용자 전용 쓰기 — 시스템 불가침
├── 2_지식/                 # ② 사용자 읽기·공유
│   ├── MOC.md · profile.md
│   ├── recent.md · open-loops.md    # 세션 간 맥락(T1)
│   └── notes/ sessions/ decisions/  # 지식·세션·결정(T2)
└── 3_시스템/               # ③ 기계 전용 (그래프 숨김)
    ├── conventions.md · config.json
    └── skills/ hooks/ _index/ _ref/ _claude-memory/
```
- **① 1_수집**: 사용자 저작물. 시스템이 **수정·삭제 절대 안 함**. 승격만(원문 보존).
- **② 2_지식**: 사용자가 보는 지식. Claude가 쌓되 사람이 읽고 공유.
- **③ 3_시스템**: 기계 인프라. 스킬·훅·인덱스·설정. 사용자가 건드릴 필요 없음.
- 시인성(누가 보나)과 논리 티어(T0/T1/T2)는 직교한다.

### 3.3 계층 로드
토큰 최적화의 핵심 원리 — **"넣은 걸 압축"보다 "애초에 안 넣기".** 수천 노트를 로드하지 않는다.
- **T0 (항상 로드)**: `CLAUDE.md`(≤~1,200토큰) + 자동 메모리 `MEMORY.md`(200줄 상한). 규약과 사용자 정체성.
- **T1 (세션 시작 주입)**: `recent.md`(최근 세션 1줄씩) + `open-loops.md`(진행 중). SessionStart 훅이 주입.
- **T2 (검색으로만 로드)**: notes·sessions·decisions·_ref의 수천 노트. **통독 금지** — 검색 top-k만, 주입 상한 ~2,000토큰.

CLAUDE.md는 5스타일 최소 코어로 고정한다: §0 사용자(인터뷰 주입) · §1 세션 시작 · §2 읽기·쓰기(통독금지·요약≠원문대체·1_수집 불가침·삭제없음) · §3 진화(one-in-one-out, 200줄 상한). 규약이 자라면 하나 넣을 때 하나 빼서(one-in-one-out) 기본 로드가 다시 비대해지지 않게 막는다.

### 3.4 git·백업 — 2-repo 모델
- **볼트 = git repo.** SessionEnd 훅이 세션마다 자동 커밋·push한다(Claude 변경분과 사용자 1_수집 변경분을 author 분리 커밋).
- **push 제외**: `sensitive: true` 노트(pre-commit이 unstage) + 파생·휘발분(`_index`·`_claude-memory`·`_변환본`·PDF, .gitignore). 인덱스·메모리는 재구축 가능하므로 백업 안 해도 비치명.
- **2-repo 모델**: **배포 repo**(모두 clone하는 템플릿) vs **개인 repo**(각 사용자의 실제 진화 볼트). 빌드·배포 중엔 `origin` = 배포 repo. setup-interview가 개인 repo URL을 받으면 `origin` → 개인 repo로 재지정하고 배포 repo는 `dist`로 보존한다. 이후 자동 push는 개인 repo로 간다. **섞이면 안 된다 — 배포판엔 개인 데이터가 0이어야 한다.**

## 4. 검색 — 측정으로 확정한 하이브리드

검색은 목적 ①(관련만 로드)·②(맥락 복원)의 핵심 엔진이다. **어떤 검색이 필요한지는 추측이 아니라 측정으로 정했다.**

### 4.1 측정 여정 (측정 전 아키텍처 금지)
1. **베이스라인**: 벡터 없이 Claude의 에이전틱 grep(MOC→grep→선별 Read)로 골든셋 hit@8 측정.
2. **함정 발견**: literal 골든셋(고유명사 공유)만으로는 grep이 100%. 그러나 이건 grep 홈그라운드만 잰 순환 측정이었다. **패러프레이즈 골든셋**(고유명사·헤딩어 제거한 의미 질의)을 추가하니 실사용에 가까운 그림이 나왔다.
3. **3자 측정 결과**:
   - literal 30문: lexical 93.3% / vector 83.3% / **hybrid 90.0%**
   - 패러프레이즈 12문: lexical 50% / vector 50% / **hybrid 58.3%** (각 arm이 서로 다른 6문을 맞춤 → RRF 합집합 7문 = 상보성 실증)
4. **결론 = 하이브리드 채택.** grep 단독·벡터 단독 모두 기각. 실사용 질의는 두루뭉술 → 벡터 필요, 그러나 grep 정밀도도 보존해야 한다.

### 4.2 as-built 구조 (`3_시스템/search.py`)
- **하이브리드 = lexical(grep) + vector RRF 융합** (Reciprocal Rank Fusion, k=60).
- **임베딩 모델 = `intfloat/multilingual-e5-small`** (384차원, CPU, ~470MB). 측정으로 채택. e5 규약대로 질의는 `query:`, 문서는 `passage:` 프리픽스.
- **하드코딩 금지**: 모델은 `config.json`의 `embed_model`에서 읽음. 티어 전환 = config 한 줄 + `--rebuild`(차원이 달라 재빌드 필수). 코드는 불변.
- **on-demand**: 데몬·포트·와처 없음. 인덱스는 `_index/`에 JSON 영속, hash 증분 재인덱싱.
- **청킹**: 헤딩 단위, max-pool. recency 가중은 제거(측정상 순노이즈), RRF가 융합.

> **왜 e5-small인가 (bge-m3 아니라)?** 설계 초기엔 GPU 티어(bge-m3, 1024차원)를 상정했다. 그러나 측정 결과 e5-small로 골든셋 목표를 통과했고, "측정 > 티어표" 원칙에 따라 최저가 구성을 채택했다. bge-m3는 폐기가 아니라 **강GPU 배포 사용자의 정규 상위 티어**로 [[#10. DEFERRED 창고|§10 DEFERRED]]에 보존.

## 5. 의미 그래프 (B1) — 플러그인 0

검색용으로 이미 계산한 벡터를 재활용해(추가 임베딩 0), 노트 사이 의미 이웃을 **위키링크로** 심는다. 그러면 **Obsidian 네이티브 그래프 뷰가 곧 의미 그래프**가 된다. 별도 시각화 플러그인 불요.

### 5.1 왜 절대 임계가 아니라 상대 top-k인가
e5-small 임베딩은 **이방성**(anisotropy)이 있다. 전 노트쌍의 max-chunk 코사인이 0.80~1.00의 좁은 원뿔에 압착된다(최근접 이웃조차 0.92+). 그래서 "코사인 ≥ 0.6" 같은 절대 스칼라 임계는 원리적으로 작동하지 못한다 — 어떤 임계든 완전 그래프(헤어볼)이거나 과편화다.
- 실측: 절대 thr=0.88 → 661엣지 헤어볼(precision 8.6%). 상호 kNN → 고립 양산. 
- **채택 = per-node top-k(k=3) union.** 노드당 이웃 상한 3, 대칭 union 렌더. 결과: avg 차수 4.5, 고립 0, 성분 2(진짜 토픽 섬). 수동 위키링크 57엣지 대비 recall ~49~51%(나머지는 신규 의미엣지 = B1의 기능).

### 5.2 관리 블록 (writer)
`search.py --link-write`가 Claude 소유 노트에 접이식 콜아웃 블록을 심는다(reading·편집 양쪽에서 마커 안 보임). **SessionEnd 훅이 매 세션 자동 실행**(reindex 후 link-write)하므로 링크는 항상 최신 — 수동 호출도 가능.
```
> [!info]- 관련 노트 %%sl%%
> [[A]] · [[B]] · [[C]]
```
- **멱등**: 마커 `%%sl%%`로 기존 블록만 교체. 수동 링크·본문 불변(안전).
- **쓰기 권한**: Claude 소유(notes·sessions·_ref)만 자동 갱신. 1_수집 사용자 저작물은 안 씀.
- **피드백 루프 가드**: 관리 블록은 파생 데이터 → `strip_managed()`가 색인 유입 차단. 안 막으면 "노트가 이웃 이름 포함 → 재색인 → 그 이웃과 유사도↑" 자기강화. reindex 후 이웃 불변 실증.
- 파라미터(`neigh_k`·`dup`·`floor`)는 `config.json`의 `graph` 블록에서 읽음(setup-interview 슬롯).

> **정적 2D 의미지도는 채택하지 않았다.** t-SNE 산점도를 시험했으나 Obsidian 네이티브 그래프뷰(줌·필터·실시간·양방향)가 명백히 우월했다 — 의미그래프의 목적은 B1 위키링크 → 네이티브 그래프로 이미 완결되므로 별도 지도는 불요.

## 6. 네이티브 메모리 통제 — 옵션 R

Claude Code의 네이티브 자동 메모리(Claude가 스스로 쌓는 노트, `~/.claude/.../memory/`)는 외부뇌와 정면으로 겹친다. 둘 다 켜면 토큰 이중 로드 + 진실 원본 2개(모순).

**해법 = 옵션 R(리다이렉트).** `settings.local.json`에 한 줄:
```json
{ "autoMemoryDirectory": "C:\\Claude_Brain\\3_시스템\\_claude-memory" }
```
- 네이티브 자동학습을 **공짜로 유지**하되, 저장 위치를 **볼트 안**으로 돌린다 → 인간 가독·git·우리 검색 인덱스에 포함. 이중 시스템 소멸.
- 자동 로드 예산 = `MEMORY.md` 첫 200줄/25KB만. 토픽 파일은 on-demand.
- **배포 시 제외**: `_claude-memory`는 개인 레이어(빌더 사실)라 `.gitignore`로 배포 repo에서 뺀다. 로컬(리다이렉트 위치)엔 유지. 메모리는 재생성 가능하므로 git 백업 포기는 비치명.
- 리다이렉트 실패 시 폴백: `autoMemoryEnabled: false`(볼트 스킬이 전담).
- **배포 온보딩**: `settings.local.json`은 gitignore(머신 절대경로)라 배포판에 없다 → **setup-interview 슬롯0(설치 직후 선행)이 신규 사용자 머신에 이 리다이렉트를 설정**한다. 안 하면 네이티브 메모리가 볼트 밖 기본 위치에 남는다.

## 7. 훅 3 — 결정론적 보증층

뿌리 ②(규약 발동이 모델 의존 → 죽은 규약)를 시정한다. 강제가 필요한 것은 규약이 아니라 **결정론적 훅**으로 보장한다.

| 훅 | 종류 | 트리거 | 동작 |
|---|---|---|---|
| **pre-commit** | git native | 커밋 | secret-scan(API 키·토큰·개인키 등 비밀 패턴) + frontmatter lint + `sensitive:true` unstage. 실패 = 커밋 차단 |
| **SessionStart** | Claude Code | 세션 시작 | recent+open-loops 주입 + 인덱스 증분 + **push(밀린 커밋 동기)** + 헬스 1줄 + (인터뷰 전) "개인화 미완료" 경고 |
| **SessionEnd** | Claude Code | 세션 종료 | 볼트 커밋(Claude/사용자 author 분리) + push → B1 링크갱신 **디태치**(비대기). **best-effort 백업**(주 커밋경로는 Stop). |
| **Stop** | Claude Code | 턴 종료 | notes/·decisions/(**재귀**, `_`접두 하위폴더 제외)의 ①프론트매터 ②MOC 등재 ③**중복 basename**(2_지식/**·_ref/**) 결정론 검증 → 위반 시 종료 차단·모델 되먹임(파일 서킷브레이커로 무한루프 차단). **위반 0시 커밋**(author 분리 — 커밋 durability를 살아있는 턴에 확보: 종료=PC셧다운이 SessionEnd를 kill해도 손실 0). 인시던트 `_ref/2026-07-11-규약단계-침묵생략`·`_ref/incident-2026-07-19-SessionEnd-커밋정지` |

구현상 치명 실측 두 가지가 배포 안정성을 좌우한다:
- **pre-commit 위치**: `.git/hooks/`는 git 추적이 안 돼 clone 시 사라진다 → 추적 위치 `3_시스템/hooks/pre-commit` + 설치 시 `git config core.hooksPath 3_시스템/hooks`(README 절차).
- **PowerShell 인코딩 2층**: `.ps1`은 UTF-8 **BOM** 저장(PS 5.1이 cp949로 오독하는 것 방지) + 데이터 읽기는 `Get-Content -Encoding UTF8`. 한글 경로(`3_시스템`)와 `{{` 슬롯이 깨지지 않게. 검증은 반드시 `powershell`(5.1)로 — pwsh 7은 UTF-8 기본이라 버그를 못 잡는다.

지능 작업(세션 요약·recent 갱신)은 훅이 아니라 session-close 스킬이 한다(Claude가 세션 흐름에서 수행). 훅은 결정론적 추출·저장·커밋·**검증(Stop)**만.

## 9. 설계 원칙

이 시스템을 관통하는 규율. 여기서 벗어나는 변경은 재검토 대상이다.

- **사용자 > 제작자.** 이건 진짜 배포 제품이다("사용자=제작자"는 폐기된 가정). 그래서 §0·profile·모델은 하드코딩이 아니라 **소프트코딩**(인터뷰/config 주입). 제작자(건희)도 첫 사용자로서 배포 경로(인터뷰)를 밟는다.
- **측정 우선 — 측정 전 아키텍처 금지.** 설계 3라운드의 주 오류가 이것이었다. 벡터검색·임베딩 모델·의미그래프 임계 전부 골든셋/실측이 결정했다. 화려한 구상을 먼저 짓지 않는다.
- **삭제 없음.** `status: archived` 마킹 + 검색 가중 페널티. (지키려던 자원=디스크가 실재하지 않았다 — 세션로그는 텍스트, 헤비 1년 ≈ 수십MB.) 자동 만료 파이프라인은 그 자체 결함을 낳아 폐기.
- **DEFERRED 창고.** *"잘려나간 설계는 사라지지 않는다 — 문서로 남고, 필요가 실증되면 그때 꺼낸다."* 정교한 설계는 폐기가 아니라 복원 대기([[#10. DEFERRED 창고|§10]]).
- **요약 ≠ 원문 대체.** 요약은 포인터+gist만. 원문은 보존.
- **골든셋 = 유일 거버넌스.** 1인 시스템에서 관료적 거버넌스 절차는 자연 작동하지 않음이 실증됐다(과대 마감 3회). 유일한 강제 = 골든셋(literal 30 + 패러프레이즈 12쌍, `한국어 질의 → 정답 노트`) hit@8 측정. **합격선 hit@8 ≥ 80%**, weekly-review가 샘플 재측정해 낙폭을 잡는 회귀 게이트.
- **_ref 생존 인프라.** 버그·이상 시 `3_시스템/_ref/`의 과거 진단(incident)을 **먼저 검색** → 해결 후 incident로 기록. 구 시스템 교훈: 쓰기만 있고 읽기 경로가 없으면 죽은 폴더가 된다 → 골든셋에 _ref 질의를 넣어 재소환이 실작동하는지 검증한다. **범위 = 외부뇌 시스템 자체의 진단 전용**(훅·검색·규약·인덱스 등). **프로젝트(<프로젝트> 등)의 incident은 프로젝트 폴더**(예 `2_지식/notes/<프로젝트>/_ref/`)에 둔다 — 시스템 생존진단과 프로젝트 버그를 섞지 않는다(2026-07-19 정리).

### 12.6 훅 스크립트 명세 (결정론층 — LLM 호출 없음)
**⚠ 인코딩 2층(배포 치명)**: ① `.ps1`은 **UTF-8 BOM** 저장(훅은 `powershell` = PS 5.1로 실행, BOM 없으면 cp949로 오독 → 한글 경로·`{{` 파싱 에러). ② 데이터 읽기는 `Get-Content -Raw -Encoding UTF8`(BOM 없는 데이터 오독 방지). 검증은 반드시 `powershell`(5.1)로 — pwsh 7은 UTF-8 기본이라 버그 못 잡음.

3. **frontmatter lint** — `.md`가 `---`로 시작 안 하면: `2_지식/`=**커밋 차단**(스키마 깨진 지식노트=실오류, Stop훅 백스톱), `1_수집/`=경고(사용자 저작물 무수정). MOC 등재 검사는 여기 없음(커밋 좌초 방지 — Stop훅 담당).

**stop-check.ps1** (Stop 훅 — 뿌리② 재적용): 턴 종료 시 `2_지식/notes/·decisions/`(**재귀** — 하위폴더 포함, `$f.DirectoryName`에 `\_`(=`_`접두 폴더) 있으면 skip) 각 .md가 ①선두 `---` 프론트매터 ②파일명이 `MOC.md`에 `[[name]]`/`[[name|`/`[[name#`로 등재 됐는지 + ③**중복 basename**(`2_지식/**`·`3_시스템/_ref/**` 전역 `Group-Object BaseName` — 위키링크 `[[name]]` 모호 방지) 검사(`-Raw -Encoding UTF8`). 위반 시 stdout에 `{"decision":"block","reason":...}` + exit 0 → 종료 차단·모델 되먹임(**공식 계약**: Stop 이벤트는 top-level `decision:"block"`+`reason`을 stdout+exit0으로 읽음. exit 2+stderr도 차단되나 reason 구조화 불가). **무한루프 차단 = 파일 서킷브레이커**: `stop_hook_active` 입력필드가 현행 Claude Code 문서에 미보장이라(있으면 즉시 exit0로 병행 처리) 의존 불가 → `_index/.stopcheck-guard`에 위반집합 SHA1+카운트 기록, **동일 위반 2회 차단 후 해제**(exit0)하되 hooks.log에 UTF-8로 미해결 잔존 기록(조용히 생략 금지 — 인시던트 교훈). 정상 경로(모델이 등재→다음 검사 통과)는 애초에 루프 안 남. MOC 없으면(초기)·notes 폴더 없으면 스킵. sessions/·_ref/·`_`접두 하위폴더는 대상 아님. 서킷 sig에 dupNames 포함(중복도 무한루프 방지). **재귀화+중복검사 = 2026-07-19 추가**(정리용 notes 하위폴더도 MOC 강제·동명노트 차단). 배경 = 인시던트 `_ref/2026-07-11-규약단계-침묵생략`(MOC 등록·프론트매터가 모델 의존층에만 있어 침묵 생략됨 → 결정론 훅으로 이관).
**+ 커밋 durability(2026-07-19 추가)**: 위반 0(clean)일 때 exit 0 전에 **커밋 수행** — `git -C add -A ':!1_수집'` → 변경 시 `commit Claude author`, `git -C add 1_수집` → 변경 시 `commit User author`. 모든 git 출력 억제(**Stop 계약: stdout은 block JSON만** — 오염 시 파싱 깨짐. 실증: clean 시 STDOUT 공백). 무변경 시 diff 가드로 무커밋(토큰·커밋 0). 커밋 후 스테이지 잔존(=pre-commit 차단) 시 hooks.log 기록(무성 좌초 가시화). **이유**: 실제 종료가 PC 셧다운이라 SessionEnd가 OS에 kill돼 커밋 완주 불가 → 커밋을 살아있는 턴(Stop)으로 이관(torch 미로드라 ~수백ms). 배경 = `_ref/incident-2026-07-19-SessionEnd-커밋정지`.

**session-start.ps1**: ① hooks.log에 발화 기록 → ② CLAUDE.md에 미치환 실슬롯(`\{\{[^}]` — 문서행 리터럴 `{{}}` 제외) 있으면 "개인화 미완료" 출력 → ③ recent.md·open-loops.md 내용 stdout(`-Raw -Encoding UTF8`, 컨텍스트 주입) → ④ 헬스 1줄 `[뇌] 노트 N개 | 인덱스 age Xd | 마지막커밋 Y`(노트 N = `notes·sessions·decisions`만 재귀 카운트) → ⑤ search.py 있으면 `py -3 search.py --reindex` → ⑥ **push(2026-07-19 추가)**: 개인화 완료(미치환 슬롯 없음)+원격 존재 시 `git -C push`(무출력·실패 무시). **이유**: 셧다운이 SessionEnd를 kill해 push 못 해도 **다음 세션 시작 때 밀린 커밋을 확실히 원격 동기**(셧다운 견고).

**session-end.ps1** (요약·recent 갱신은 session-close 스킬. **2026-07-19 재구성** — 아래 순서 변경·ML 디태치): ① hooks.log 기록 → ② `git add -A -- ':!1_수집'` → 변경 시 `commit --author="Claude <claude@local>"`(실패=pre-commit 차단 시 hooks.log) → ③ `git add 1_수집` → 변경 시 `commit --author="User <user@local>"`(author 분리) → ④ **원격 있고 개인화 완료 시**(미치환 슬롯 없을 때만) `git push`(실패 무시. 개인화 전엔 origin이 공개 배포 repo일 수 있어 억제 — 슬롯9 재지정 후 활성) → ⑤ **B1 링크 갱신 디태치**: `Start-Process py --reindex --link-write -WindowStyle Hidden`(비대기). **구 구조는 ②의 reindex+link-write를 커밋 앞에서 동기 실행 → 콜드 세션서 훅 타임아웃(60s) 전에 kill → 커밋 미도달(한 주 정지)**. 시정: 커밋은 Stop 훅(턴마다)이 주담당·여기선 best-effort 백업, ML은 맨 끝 디태치로 훅을 **절대 블록 안 함**(타임아웃 무의미화). reindex는 SessionStart도 수행(중복 무해), link-write=멱등·1세션 지연 무해·인덱스는 gitignore. 배경 = `_ref/incident-2026-07-19-SessionEnd-커밋정지`.

### 12.9 search.py — 하이브리드 검색 + B1 (핵심 재구축 대상)
실물 = `3_시스템/search.py`. stdlib만 상단 임포트, `sentence_transformers`·`torch`는 지연 임포트. **stdout/stderr `reconfigure(encoding="utf-8")`**(Windows 콘솔 cp949 → `—` 등 출력 크래시 방지).

**CLI**: `--reindex`(증분) · `--rebuild`(전체) · `--q "질의" --k 8 --mode lexical|vector|hybrid` · `--neighbors <path>`(B1) · `--eval golden.jsonl`(3자 hit@k) · `--link-dryrun`/`--link-write`(B1 writer).

**인덱스** `_index/embeddings.json`: `{"_model":<name>, "<상대경로>":{hash(sha1),mtime,status,gist,vecs:[[384float],…]}}`. 대상 글롭 = `2_지식/**`(설계노트·사용자설명서 포함) · `3_시스템/_ref/**` · `_claude-memory/**` · `_index/pdf-cache/**`. 증분 = hash 비교. **모델 각인**: config 모델 ≠ 인덱스 `_model`이면 `--reindex`도 자동 전체 재빌드(차원혼합 가드).

**청킹** (`chunk_text`): frontmatter 제거 → 헤딩 경계로 섹션 분할 → 섹션이 `WINDOW=40`줄 초과면 창 분할. 노트당 `MAX_CHUNKS=120` 상한. 노트 스코어 = **max-chunk cosine**(통짜 희석 방지).

**어휘 arm** (`lexical_scores`): 질의 토큰화(공백·구두점 split, 한국어 조사 접미 제거 `은/는/이/가/…`, 2자+, 소문자, 중복제거) → 노트 본문(`strip_managed` 후 소문자)에서 `score = 매칭토큰수*10 + log1p(총출현빈도)`. grep형.

**벡터 arm** (`vector_scores`): 질의 임베딩(e5=`query:` 프리픽스, bge=생략) → 각 노트 max-chunk cosine. `RECENCY_W=0.0`(recency 역효과 진단 → off). archived ×0.5.

**융합**: `hybrid = RRF(lexical_rank, vector_rank)`, `RRF_K=60`. lexical은 매칭>0 노트만(0점 순위노이즈 배제). RRF 점수 = Σ 1/(k+rank+1).

**모델 티어 전환**(코드 불가지): config 한 줄 교체 + `--rebuild`. 저=e5-small(384,CPU,기본) / 중=e5-base(768) / 고=bge-m3(1024,GPU fp16, `pip install torch --index-url …cu121`). e5=프리픽스, bge=`is_bge`로 생략.

**B1 이웃** (`neighbors`, §5.1 근거): per-node top-k **상대**방식(절대임계 폐기). 파라미터 = `config.json "graph"`(기본 `neigh_k=3`·`neigh_dup=0.999`·`neigh_floor=0.08`). 알고리즘: 타깃 벡터 vs 전 노트 max-chunk cos → `cos≥dup`(색인중복) 배제 → 내림차순 top-k → `cos < top1 - floor`면 중단(상대 하한).

**B1 writer** (`write_links`): 스코프 `WRITER_GLOBS` = `2_지식/notes·sessions` + `3_시스템/_ref`(Claude 소유만, 1_수집 제외). 링크대상 제외 `LINK_TARGET_EXCLUDE` = `pdf-cache/`·시스템문서(`설계노트`·`사용자설명서` — 검색은 되나 지식그래프 링크 대상서 제외). basename dedup. 블록 = 접이식 콜아웃 `> [!info]- 관련 노트 %%sl%%\n> [[A]] · [[B]] · [[C]]`. 멱등(마커 `%%sl%%`로 기존 블록 교체, `_BLOCK_ALT`가 v1/v2/v3 3세대 매치). **`strip_managed`**: 관리블록은 파생데이터 → 색인(임베딩+lexical) 유입 차단(이웃이름 피드백루프 방지).
