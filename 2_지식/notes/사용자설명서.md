# 외부뇌 사용자설명서

외부뇌는 Obsidian 볼트 + Claude Code로 만든 개인 지식 시스템이다. 목적은 셋 — **토큰 아끼기 · 맥락 되살리기 · 답변 질 높이기**.
이 문서는 "어떻게 쓰나"를 짧게 설명한다. "왜 이렇게 만들었나"가 궁금하면 각 항목의 → [[설계노트]] 링크로 넘어가면 된다.

> 설치는 [README](../../README.md)를 따르세요. 이 문서는 설치가 끝난 뒤 읽는 사용 안내입니다.

---

## 처음 할 일 — 개인화 (딱 한 번)

첫 세션을 열면 "⚠ 개인화 미완료" 안내가 뜬다. 그때 이렇게 입력하세요:

```
setup-interview
```

~10분 대화로 호칭·언어·톤·하드웨어(임베딩 모델 자동 결정)·백업 repo를 잡는다. 끝나면 경고가 사라지고, 이후엔 그냥 대화하면 된다. 나중에 다시 맞추고 싶으면 `/재보정`.
→ 무엇을 채우는지: [[설계노트#8. 스킬 8개]] · 왜 인터뷰인지: [[설계노트#9. 설계 원칙]]

## 파일 계층도 — 무엇이 무엇인가

볼트를 열면 폴더 셋(`1_수집`·`2_지식`·`3_시스템`)과 루트 README가 보인다. **누가 읽고 쓰느냐**로 나뉜다. 아래가 전부다.

```
Claude_Brain/
├── CLAUDE.md            # [시스템] Claude가 매 세션 읽는 규약. 건드리지 마세요
├── README.md            # 설치 안내
│
├── 1_수집/              # ① 내가 쓰는 곳 — 시스템이 절대 손대지 않음
│   └── _inbox/          #    떠오르는 것 아무렇게나 던져두는 수집함
│
├── 2_지식/              # ② 뇌가 쌓는 지식 — 내가 읽고 공유
│   ├── MOC.md           #    지식 지도(목차). 어디에 뭐가 있나
│   ├── profile.md       #    내 프로필(인터뷰가 채움). 호칭·톤·경계
│   ├── recent.md        #    최근 세션 요약(자동). 세션 시작 시 되살아남
│   ├── open-loops.md    #    진행 중인 일(자동). 다음에 뭘 하려 했는지
│   ├── notes/           #    지식 노트 — [[설계노트]]·[[사용자설명서]](이 문서)가 여기
│   ├── sessions/        #    지난 세션 요약 보관
│   └── decisions/       #    내린 결정 기록(번복해도 남김)
│
└── 3_시스템/            # ③ 기계 — 건드릴 필요 없음
    ├── conventions.md   #    노트 규격(프론트매터 스펙)
    ├── config.json      #    설정. 임베딩 모델·그래프 이웃 수
    ├── search.py        #    검색 엔진(하이브리드)
    ├── hooks/           #    자동 훅(커밋·secret검사·세션 시작/턴/종료)
    ├── _index/          #    검색 인덱스(자동 생성, 백업 안 함)
    ├── _ref/            #    외부뇌 시스템 문제 진단(버그 나면 여기 먼저 봄. 프로젝트 버그는 프로젝트 폴더로)
    ├── _eval/           #    골든셋·검색 성능 측정
    └── _claude-memory/  #    Claude 자동 메모리(볼트 안으로 리다이렉트)
```

숨은 폴더도 있다(건드릴 일 없음): `.claude/`(스킬 8개 + 훅 설정) · `.obsidian/`(Obsidian 설정 + Dataview 플러그인).

**기억할 규칙은 하나뿐**: `1_수집/`은 내 것(시스템 불가침), `2_지식/`은 함께 보는 지식, `3_시스템/`은 기계. 나머지는 알아서 돌아간다.
→ 왜 이렇게 나눴나: [[설계노트#3.2 폴더 구조 — 시인성 3분류]]

## 일상 사용

대부분은 **그냥 평소처럼 대화**하면 된다. 시스템이 알아서 맥락을 되살리고 지식을 쌓는다. 필요할 때 쓰는 것들:

| 하고 싶은 것 | 이렇게 | 더 알기 |
|---|---|---|
| **볼트에서 찾기** | "X에 대해 볼트에서 찾아줘" (또는 `search`) — 통독 대신 관련 노트 top-k만 | [[설계노트#4. 검색 — 측정으로 확정한 하이브리드]] |
| **PDF 넣기** | "이 PDF 넣어줘" — 통째 읽지 않고 요지+검색용으로 흡수 | [[설계노트#8. 스킬 8개]] |
| **아이디어 모으기** | `1_수집/_inbox/`에 자유롭게 적어두기 → `inbox-sort`가 지식으로 승격 | [[설계노트#3.2 폴더 구조 — 시인성 3분류]] |
| **세션 마무리** | "세션 종료" — 요약·recent·open-loops 정리 | [[설계노트#8. 스킬 8개]] |
| **주간 점검** | `weekly-review` (주 1회) | [[설계노트#8. 스킬 8개]] |

세션을 새로 열 때마다 최근 흐름(recent)과 진행 중인 일(open-loops)이 자동으로 되살아난다. 별도로 뭘 할 필요 없다.
→ 원리: [[설계노트#3.3 계층 로드]]

## 사용법 예시

실제로 어떻게 대화하는지. `>`는 내가 입력하는 것, 아래는 시스템이 하는 일.

**예시 1 — 지식 찾기 (통독 없이)**
```
> 예전에 정리한 컨텍스트 압축 관련 내용 찾아줘
```
→ 시스템이 폴더를 통째로 읽지 않는다. grep + 의미검색으로 관련 노트 몇 개만 골라 경로와 요지를 준다. "이 노트에 정리돼 있고, 요지는 …". 토큰을 아끼는 핵심 동작.

**예시 2 — PDF를 지식으로**
```
> 이 논문 PDF 넣어줘 (파일 경로 첨부)
```
→ 통째로 읽어 토큰을 태우지 않는다. 변환 → 요지 통찰노트(`2_지식/notes/`) 생성 → 검색용 인덱싱. 이후 "그 논문에서 X는?" 하면 검색으로 해당 부분만 꺼낸다.

**예시 3 — 떠오른 생각 모았다가 승격**
```
(1_수집/_inbox/ 에 메모를 여러 개 적어둠)
> inbox 정리해줘
```
→ `inbox-sort`가 각 메모를 분류하고, 지식은 **원문 그대로** `2_지식/notes/`로 승격(요약해서 버리지 않음). 원본 수집물은 손대지 않는다.

**예시 4 — 세션 끝낼 때**
```
> 세션 종료
```
→ 이번 대화를 요약해 `sessions/`에 저장, `recent.md`에 한 줄, `open-loops.md`에 다음 할 일 기록. 커밋은 이미 **턴마다 자동**(살아있는 세션 중 — PC를 그냥 꺼도 안전), 원격 백업 push는 세션 시작·종료에 동기된다. 다음에 열면 이 맥락이 되살아난다.

**예시 5 — 다음 날 새 세션 (아무것도 안 해도 됨)**
```
> (그냥 새 세션을 연다)
```
→ "최근 이런 작업을 했고, 진행 중인 건 이것" 하고 맥락이 자동 주입된다. 긴 설명을 다시 붙일 필요가 없다. 이게 맥락 복원.

## 사용 꿀팁

시스템을 제대로 뽑아 쓰는 요령.

- **두루뭉술하게 물어도 된다.** 정확한 제목·단어를 몰라도 의미검색이 잡는다. "왜 그때 그렇게 정했더라" 같은 말도 통한다. 단어를 정확히 떠올리려 애쓰지 마라.
- **세션 종료를 습관으로.** "세션 종료"를 해야 요약·recent·open-loops가 남아 다음 세션 맥락이 이어진다. 안 하면 다음에 처음부터 설명하게 된다.
- **_inbox엔 날것을 던져라.** 정리·형식은 신경 쓰지 마라. 나중에 `inbox-sort`가 원문 그대로 지식으로 승격한다. "나중에 정리" 부담 없이 일단 담기.
- **결정은 남겨라.** 뭔가 정했으면 "이거 decisions에 기록해줘". 몇 달 뒤 "왜 이렇게 했지?"가 검색으로 바로 나온다.
- **PDF는 넣고 검색으로.** "이 PDF 다 읽고 요약해"는 토큰을 태운다. "넣어줘" → 이후 필요한 부분만 검색. 논문·문서가 쌓일수록 이득.
- **중요한 노트는 눈에 띄게.** 프론트매터 `importance`를 높이면(1~10) 검색 상위에 온다. 자주 참조할 핵심만.
- **그래프 뷰를 종종 열어라.** 시스템이 이어준 의미 링크로 "잊고 있던 연결"이 보인다. 생각이 막힐 때 특히.
- **버그는 되묻기.** "예전에 비슷한 문제 어떻게 풀었지?" 하면 `_ref`의 과거 진단을 재활용한다. 같은 삽질 반복 안 함.
- **주 1회 `weekly-review`.** 흩어진 세션이 통찰노트로 응축되고, 검색 성능도 자동 점검된다. 손이 안 가게 하려면 인터뷰(또는 `/재보정`)에서 **클라우드 루틴 cron 자동 실행**을 걸 수 있다(개인 백업 repo 연결 필요).
- **톤이 안 맞으면 `/재보정`.** 너무 장황하거나 딱딱하면 언제든 다시 맞춘다. 개인화는 고정이 아니다.

## 알아두면 좋은 것

- **민감한 노트**는 프론트매터에 `sensitive: true` → git push에서 제외된다.
- **삭제는 안 한다.** 안 쓰는 노트는 지우는 대신 `status: archived`로 마킹(검색에서 가중 낮춤). → [[설계노트#9. 설계 원칙]]
- **비밀 차단**: API 키 같은 게 커밋에 섞이면 자동으로 막힌다(pre-commit 훅). → [[설계노트#7. 훅 3 — 결정론적 보증층]]
- **지식노트 자동 보장**: `notes/`·`decisions/`에 새 지식노트가 생기면 프론트매터와 MOC 등재가 자동 강제된다(Stop 훅, **하위폴더까지 재귀**). 누락되면 턴을 끝낼 때 시스템이 스스로 막고 채운다 — 지식이 지도에서 미아가 되지 않게. **검증을 통과하면 그 턴에 바로 커밋**(그래서 PC를 갑자기 꺼도 작업이 안 날아간다). → [[설계노트#7. 훅 3 — 결정론적 보증층]]
- **폴더로 정리해도 안전**: `notes/` 안에 새 폴더를 만들어 노트를 묶어도 된다(검색·MOC 강제 그대로 미침). 세 가지만 지켜라 — ① 프론트매터 유지 ② **파일명은 볼트 전역 고유**(같은 이름 노트 2개면 위키링크 `[[이름]]`이 헷갈려 → Stop 훅이 막는다) ③ 핵심파일(MOC·recent·open-loops·설계노트·사용자설명서)·특수 `_`폴더는 그대로 둔다. → [[conventions#파일명·폴더 규약 (Stop 훅이 강제)]]
- **백업**: 커밋은 턴마다 자동, 원격 push는 세션 시작·종료에 best-effort 동기(PC 셧다운으로 종료 훅이 못 돌아도 다음 시작 때 밀린 커밋을 올린다). 개인 비공개 repo를 인터뷰에서 연결했다면 거기로 간다. → [[설계노트#3.4 git·백업 — 2-repo 모델]]

## 문제가 생기면

- **버그·이상**: 시스템이 `3_시스템/_ref/`의 과거 진단을 먼저 찾아본다(같은 문제 재활용). 해결 후엔 거기 기록한다.
- **검색이 이상**: 인덱스는 파생물이라 언제든 재구축 가능. `py -3 3_시스템/search.py --rebuild`.
- **더 깊이**: 전체 설계 맥락은 [[설계노트]] 한 문서에 다 있다.
